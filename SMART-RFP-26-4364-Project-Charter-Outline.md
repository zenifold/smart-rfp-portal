# Project Charter Outline
## RFP 26-4364 - ERP System Upgrade Consultant

**Purpose:** This document provides the outline for the Project Charter, a critical Phase 1 deliverable that establishes the foundation for SMART's ERP System Upgrade project.

---

## Overview

The Project Charter is the formal authorization document for SMART's ERP System Upgrade project. It serves as:
- **Formal Authorization**: SMART executive team approval to proceed
- **Project Foundation**: Basis for all project planning and execution
- **Stakeholder Agreement**: Shared understanding of project scope, objectives, and approach
- **Reference Document**: Go-to document for project questions and decisions

**When Delivered:** Month 1 (Week 2-3)  
**Approved By:** SMART Executive Team and Joint Venture Partners

- Contract terms negotiation

**Phase 6: Implementation Oversight (Months 10-24)**
- Project charter and governance establishment
- Integration architecture design
- Data cleansing and migration oversight

- Guardian oversight throughout vendor execution
- Go-live support
- Hypercare support (Months 21-22)
- Knowledge transfer and closeout

**4.2 Out-of-Scope Activities**

**Explicitly Excluded:**
- ERP software licensing costs (vendor responsibility)
- Hardware infrastructure upgrades (SMART responsibility, vendor recommendations provided)
- Ongoing ERP system maintenance post-implementation (vendor responsibility)
- Custom software development beyond vendor standard features
- Business process reengineering beyond workflow optimization
- Data migration execution (vendor responsibility, Guardian oversight only)
- User training delivery (vendor responsibility with Guardian oversight)
- Post-implementation ERP system enhancements (separate project)

**4.3 Scope Management Process**

**Change Request Process:**
1. **Request Identification**: Any stakeholder identifies potential scope change
2. **Impact Analysis**: Guardian analyzes impact on timeline, budget, objectives
3. **Recommendation**: Guardian provides SMART with recommendation (approve/reject/modify)
4. **SMART Decision**: SMART executive team makes final decision
5. **Documentation**: Approved changes documented in Project Charter
6. **Communication**: Stakeholders notified of scope changes

**Scope Creep Prevention:**
- Guardian reviews all vendor-proposed changes before SMART consideration
- Change requests require business justification and ROI analysis
- SMART executive approval required for any scope changes
- Budget impact documented before approval
- Timeline impact documented before approval

## Project Charter Preparation Process

### Before Charter Development

- [ ] SMART RFP 26-4364 requirements
- [ ] Pre-proposal conference notes (if attended)

### Charter Development (Month 1, Week 1-2)

**Activities:**
1. Draft executive summary based on RFP requirements
2. Document business justification using SMART's challenges
3. Define SMART objectives based on RFP success criteria
4. Establish project scope with SMART stakeholders
5. List deliverables by phase

7. Document firm-fixed pricing structure
8. Establish governance structure with SMART
9. Identify top 5 risks and mitigation strategies
10. Define quality standards and assurance processes
11. Establish success criteria (SMART-validated)
12. Prepare approval section for signatures

### Charter Review (Month 1, Week 3)

**Review Process:**
1. **Internal Review**: Consultant team reviews draft charter
2. **SMART Review**: SMART stakeholders review and provide feedback
3. **Revisions**: Consultant incorporates SMART feedback
4. **Final Review**: SMART approves final version

### Charter Approval (Month 1, Week 4)

**Approval Process:**
1. **SMART Executive Approval**: SMART executives review and sign
2. **Joint Venture Approval**: BBC, LLC and Maximilian Murphy sign
3. **Distribution**: Approved charter distributed to all stakeholders
4. **Project Initiation**: Project formally kicks off per charter
